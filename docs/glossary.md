# Glossary

Terms used in the GRID//NODE handbook. Each definition is plain English + a one-line technical gloss.

---

## A

**ADR (Architecture Decision Record)**
A short doc explaining a design decision: what was decided, why, what alternatives were considered, what the consequences are. See `docs/decisions/`.

**Audit formula (Phase A→D)**
The process for fixing duplicate functions safely:
1. Extract function bodies with brace-depth tracking
2. Two-tier normalization (exact + cosmetic)
3. Classify as identical / cosmetic / true drift
4. Only merge identical (with scope check)

**Audit tag**
One of: `delete:`, `stdlib:`, `native:`, `yagni:`, `shrink:`. Used in Ponytail audits and the Flex Directive to tag findings.

---

## B

**Baseline**
The locked reference file for the project. Currently `gridnode-v1.3_post-phase-D_baseline.html`, SHA `7b6c4dc9...`. Recorded in `baseline.sha`.

**Boot fix**
A specific fix that wrapped `loadApp()` and `showScreen('app')` calls in try/catch to handle the case where they throw. The result was +111 bytes from the previous locked version.

---

## C

**CE**
Not used. (Sometimes confused with "Claude Engineer" — that's Claude in the VEKTOR role, not "CE.")

**Cloudflare Pages**
The hosting platform for the GRID//NODE app. URL: `https://gridnode.pages.dev`. Account ID: `f008e0b7e3867a6050b412d931a9abd9`.

**Consolidation review**
A script that audits a baseline for over-engineering. Outputs a report (does NOT apply fixes). Run when growth exceeds 10% between baselines (Rule 4).

**Count forward and backward**
The verification rule: totals must reconcile. E.g., if you remove 57 duplicate function definitions, the count should go from 57+17 to 17+0 (canonical count). The math closes.

---

## D

**Derived artifact**
A file that's generated by a script rather than maintained by hand. The protected-keyword list is a derived artifact. The advantage: it can't lie or drift.

**Diff math**
The discipline of verifying a byte-level diff before shipping. `original_bytes + delta_bytes = new_bytes`. Run `wc -c` on both files before and after a change.

**Drift**
A change where two pieces of code look the same but aren't (scope differences, IIFE membership, etc.). Drift merges cause runtime errors even though the diff looks clean.

---

## E

**Effectiveness Methodology v1**
The 7 verification habits from the GRID//NODE work. Saved in memory topic + `docs/decisions/`. Includes "self-test on a known case before scaling," "count forward and backward," etc.

**Exact match**
A keyword match where the substring is anchored (not just contained). The keyword `play` should match `function play(` and `\bplay\(`, NOT bare `play` substring (which would match `display:` 517 times).

---

## F

**Flex Directive v5**
The 3-lane change policy co-developed with Claude. GREEN (<1KB, no protected touch), YELLOW (medium, self-audit + 1 review), RED (protected, per-change human sign-off).

**Founder HQ**
Pipe. The human who has final authority over the project. Founder HQ signs off on RED changes and ratifies methodology changes.

---

## G

**GREEN lane**
A change that is additive, reversible, no protected-system touch, <1KB net change. Self-verifies with the smallest runnable check + the protected-keyword gate. No human sign-off.

**GRID//NODE**
The private biotech tracker app. Single-file PWA, 1MB. Lives at `https://gridnode.pages.dev`. Source code in private repos.

---

## H

**Handoff doc**
`GRIDNODE_HANDOFF.md`. The single source of truth for picking up the work in a new session. Lives in the repo (permanent).

**Hoisted function**
A function moved from an IIFE to the shared global scope. Phase A→D hoisted 17 functions: `safeText`, `page`, `stage`, `getHudElements`, `activeSelectedZone`, `qs`, `qsa`, `esc`, `norm`, `play`, `parseTimeValue`, `scrollSelectedPanelIntoView`, `formatDateForInput`, `formatDateTime`, `normalizedSite`, `validDate`, `validDateParts`.

---

## I

**IIFE**
Immediately-Invoked Function Expression. A pattern like `(function() { ... })()` that creates a private scope. Used in the GRID//NODE baseline to isolate modules.

**IIFE-scoping**
A discipline concern: a function defined inside an IIFE is NOT accessible outside it. If two IIFEs both define `function foo()`, they're independent. Hoisting to shared scope can break cross-IIFE references.

---

## K

**Keyword list**
The 130 protected identifiers generated by `scripts/keyword-extractor.js`. Includes scanner core, boot lifecycle, vault, localStorage, Phase Engine, RESULTS, WEIGHT RECORDS, SHOT CRUD/HISTORY, NODE ALIAS, hoisted shared utilities, and window.gn* globals.

**Keyword gate**
The script `scripts/protected-keyword-gate.js`. Takes a baseline + a diff, exits 0 if clean or 1 if any protected keyword is touched. Used to enforce RED classification automatically.

---

## L

**Lane**
GREEN, YELLOW, or RED. Every change is classified into one before work starts.

**Live URL**
`https://gridnode.pages.dev`. Where the deployed GRID//NODE app runs. TinyURL: `https://tinyurl.com/25h4qg7x`.

**Lock (a baseline)**
The act of declaring a specific file + SHA as the canonical reference for the project. Future changes are diffed against the locked baseline.

**Mavin**
The active builder instance. M3 model, Mavis harness. The "I" of this conversation. The other AI in Pipe's stack.

**Memory topic**
A persistent note in Mavis's memory system. Survives a single session but dies when the session ends. Different from the GitHub repo (which is permanent).

**Methodology stack**

## P

**Phase A→D consolidation**
The work that produced the current locked baseline. Phase A: trace. Phase B: implement. Phase C: staging. Phase D: report. 17 functions hoisted, IIFE unwrapped, patch comments stripped.

**Ponytail**
The lazy-senior-dev mode skill from `DietrichGebert/ponytail` (MIT). Six skills: `ponytail`, `ponytail-audit`, `ponytail-review`, `ponytail-debt`, `ponytail-gain`, `ponytail-help`. Default mode: `full`.

**Ponytail: comment convention**
The discipline of marking intentional simplifications with `// ponytail: <ceiling>, <upgrade path>`. Used by `ponytail-debt` to track deferred work.

**Protected keyword**
Any of the 130 identifiers in `scripts/PROTECTED_KEYWORDS.js`. Touching a protected keyword forces the change to RED classification.

**Protected system**
One of the 14 systems in `protected-systems.md`. Examples: SHOTS scanner, Phase Engine, VAULT, etc. Touching a protected system = RED change.

---

## R

**RED lane**
A change that touches a protected system, involves drift, consolidates scripts/styles, or anything that broke before. Requires per-change human sign-off, trace first, staging before live, count forward and backward.

**Run-or-throw**
A test pattern where the test runs a command and asserts on the output, accepting that the command may exit non-zero (e.g., when exit code = count).

---

## S

**SHA-256**
The cryptographic hash of a file. Used as the canonical identifier for the locked baseline. The current baseline SHA is `7b6c4dc9025aa07bb336edd0eb28cf76a3bd14cff7393aa19a01cb26428e6660`.

**Self-check**
The Ponytail "smallest runnable check" rule. Non-trivial logic ships with a check that's the smallest thing that fails if the logic breaks.

**Sign-off (per-change)**
The Founder HQ (Pipe) explicitly approving a RED change before it's merged. Recorded in the standing report.

**Standing report**
The block at the end of every change documenting: lane, before/after bytes+SHA, protected systems touched, dup functions introduced, self-check result, why-this-lane sentence, tested-by, claims verified vs unverified.

---

## T

**Third-signal check**
The discipline that any canonical artifact must be verified by a party other than its author + the verifier. Two AIs cannot be each other's final check. Proven by 3 rounds of evidence (v3 keyword list, v4 count, v5 script).

**TinyURL**
`https://tinyurl.com/25h4qg7x`. A short URL for the live GRID//NODE app.

---

## V

**VEKTOR**
The auditor/gate role in Pipe's stack. Filled by Claude (Anthropic). Not the same as Claude the product — VEKTOR is the role, Claude is the model.

**Verify-all**
The script `scripts/verify-all.sh`. Runs 9 checks (Node version, vitest, bash syntax, JS syntax, keyword list, gate, consolidation, markdown structure, methodology references) and reports pass/fail.

---

## Y

**YELLOW lane**
A change that is a new feature, touches shared scope, 1-5KB, OR reads from a protected system without modifying it. Self-audit + one verification pass + staging + deploy. No Founder HQ sign-off unless it touches a protected boundary.
